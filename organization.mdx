## Structuring Content

You organize documentation hierarchically in Gusronk to maintain clarity. Spaces contain docs and sub-spaces, forming a tree-like structure. This approach prevents clutter and aids navigation.

<Callout kind="success">
  Consistent naming conventions improve findability.
</Callout>

Begin with top-level spaces for major projects. Within each, create folders for categories like 'API', 'User Guides'. Assign permissions at space or doc level.

## Tagging and Search

Apply tags to docs for cross-referencing. Gusronk's search indexes tags, content, and metadata.

<Tabs>
  <Tab title="Adding Tags" icon="tag">
    In doc settings, enter comma-separated tags.
    ```javascript
    // Example tag application via API
    const doc = { id: 123, tags: ['api', 'v1', 'internal'] };
    fetch(`/api/docs/${doc.id}`, {
      method: 'PATCH',
      body: JSON.stringify(doc)
    });
    ```
  </Tab>
  <Tab title="Advanced Search" icon="search">
    Use operators like `tag:api` or `space:project`.
  </Tab>
</Tabs>

You filter results by date or author. Bookmarks save frequent searches.

<Steps>
  <Step title="Create Folder" icon="folder">
    Right-click in sidebar and select 'New Folder'.
  </Step>
  <Step title="Move Docs" icon="move">
    Drag docs between folders or spaces.
    ```bash
    # CLI equivalent
gusronk move doc/123 folder/api
    ```
  </Step>
  <Step title="Set Permissions" icon="shield">
    Edit access for viewers, editors, admins.
  </Step>
</Steps>
